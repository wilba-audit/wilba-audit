# Borrow Before You Build

**Before we build ANYTHING custom, we check the library. 80% of the time, someone has already solved the problem.**

---

## The Principle

This is one of our core principles at WILBA. For each item on your Task Audit, we ask:

1. **Is there an off-the-shelf module for this?** Check the Module Library. If there's a module that solves the problem, we grab it, install it, done.
2. **Has someone in the community built something similar?** Chances are someone has faced the same problem and has a solution or starting point.
3. **Only AFTER exhausting the library and community** do we go to custom building.

---

## Why This Matters

Building from scratch is slow and fragile. You spend 3 days building something that a pre-built module solves in 30 minutes. Modules are battle-tested, documented, and designed for reliability. They've been refined through real use.

Every module we borrow instead of building is time saved. Every time saved is bandwidth recovered. That IS the point of this whole system.

---

## The Workflow

1. Open your Task Audit
2. Look at your highest-priority items
3. Go through the Module Library — match modules to tasks
4. Install the ones that solve your problems
5. Come back to the Task Audit and tick off what's been handled
6. Whatever's left — that's what we build custom
